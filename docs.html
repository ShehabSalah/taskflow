TaskFlow User Guide

Learn how TaskFlow works — workspaces, collections, milestones, releases, time tracking, and reports. This guide mirrors the in-app Help and describes the application through version 1.2.0 (desktop).

TaskFlow runs locally on your machine. The same guide is available inside the app under Help in the account menu.

Getting started

TaskFlow helps you track time, earnings, projects, and tasks on your own machine. Data stays local — your salary and reports are not sent to the cloud.

  • Install TaskFlow for Windows or macOS from the download page. The setup wizard can set up PostgreSQL for you (Easy mode) or connect to your own database (Advanced mode).
  • Desktop updates are manual: open Settings → Desktop → Updates, check for a newer version, then use Download update to open the website releases page and run the new installer over your current app.
  • On a new computer, the wizard can restore from an encrypted backup before you create an account — skip if this is a fresh start.
  • On first launch (or after restore without sign-in), create a username and password to protect your data on this device.
  • After sign-in, pick a workspace from the top bar. Most pages use the active workspace.
  • Use the sidebar for Dashboard, Workspaces, Projects, Tasks, Logs, Reports, and Overtime.
  • Open Settings, Help, Lock, or Logout from your avatar menu (top right).

Workspaces

A workspace represents a job, client, or employer. Salary and hour expectations drive earnings calculations.

  • Create a workspace from Workspaces → New workspace. Set salary, currency, expected monthly/weekly hours, and work type.
  • Enable overtime and choose a multiplier if your workspace pays extra for overtime hours.
  • Open a workspace to see overview stats, resources, settings, and statistics tabs.
  • Start or end work from the workspace header — this runs the timer for that workspace.
  • Set a default workspace under Settings so the app opens with your usual context.

Projects & tasks

Organize work under projects, then manage tasks on a Kanban board with lists, labels, dates, checklists, and attachments. Use collections to group related work inside a project.

  • Create projects inside a workspace. Each project gets its own board lists (columns).
  • Open Tasks, pick a project, then choose a collection (or General) in the second sidebar — the Kanban shows that slice of work.
  • Drag cards between lists. Quick-add creates tasks in the current list and collection.
  • Click a task card for the quick drawer, or open the full editor for descriptions, comments, activity, and checklists.
  • Use labels, start/end dates, custom fields, attachments, and task dependencies from the task editor or drawer where available.
  • Start the timer from a task card or from the start-work dialog to link time to that task.
  • On a project detail page, use the Milestones and Releases tabs for planning targets and shipped versions (see sections below).

Collections

A collection is a planning container for related work inside one project — for example a feature area or workstream. Kanban stays the execution board; collections decide which tasks you are looking at.

  • Collections belong to a project (not the whole workspace). Switching projects shows a different collection list.
  • On Tasks, the second sidebar lists General plus your collections, grouped by status: In progress, Planned, Complete.
  • General holds tasks with no collection — existing work stays there after upgrade. You cannot delete or reorder General.
  • Create or edit a collection from the sidebar (New collection). Set name, optional description and vision, status, and optional dependencies on other collections in the same project.
  • New tasks inherit the collection you have selected. You can change collection when creating or editing a task (same project only).
  • Collection dependencies show precedence (this work waits on that area). The Collection Dependencies panel under the board is informational — it does not auto-change status.
  • Within a collection, tasks can depend on each other (blocked by / blocks). Moving a task to another collection clears those task links.
  • Optionally link a collection to a milestone when planning toward a ship target.

Milestones

A milestone is a planning target for a project — a version or delivery goal you are working toward. It groups collections so you can see progress toward that target before anything is published.

  • Open a project → Milestones tab to list, create, and edit milestones (separate from the Releases tab).
  • Link collections to a milestone from milestone detail or when creating/editing a collection (same project only).
  • Status and progress are derived from linked collection statuses — you do not set milestone status by hand.
  • Optional start and end dates support planning; the timeline view shows milestones with progress and status on hover.
  • A milestone can exist for a long time as an unreleased target. Publishing a release (below) is a separate step.

Releases

A release is the ship record for a project: version tag, title, release notes, and optional build files. Think of it as publishing what you planned in a milestone.

  • Open a project → Releases tab (separate from Milestones) to create drafts and publish.
  • Each release has a unique tag per project (for example v1.2.0), a title, markdown notes, and Draft or Published status.
  • Optionally link one milestone from the same project so the release points at the work that shipped.
  • Attach build assets with a platform label and optional channel (any product — installers, APKs, web builds). Manage platform and channel names on the project as needed.
  • Publish when ready — that sets Published and the publish date. It does not auto-complete the linked milestone; finish collections separately.
  • Project overview shows release count and the latest published tag.

Time tracking & earnings

Recorded sessions are the source of truth for duration and pay. Earnings update in real time while the timer runs.

  • Start work from a workspace detail page or the top-bar timer area. Stop when you finish.
  • The top bar shows elapsed time and estimated earnings for the active session.
  • Earnings use your workspace salary divided by expected monthly hours — stored precisely by the second.
  • If you lock or log out, the app tries to stop an active timer first so sessions stay accurate.

Dashboard

The home page summarizes the selected month for your active workspace.

  • Use the month picker to review past months. Current-month views refresh when you start or stop the timer.
  • Summary cards show worked hours, expected hours, overtime, attendance days, and earnings.
  • The weekly breakdown lists four weeks with progress toward your weekly target.
  • The workspace grid shows projects and recent tasks. You can add a task without leaving the dashboard.

Work logs

Work logs list completed time sessions with filters for workspace, project, task, and date range.

  • Open Work Logs from the sidebar to browse and filter historical sessions.
  • Duration and earnings are shown per row; totals appear for the current page.
  • Use work logs to verify what was recorded before exporting reports or timesheets.

Reports & insights

Generate files for HR, clients, or your own records. Insights show where your time went.

  • Reports → Overview: pick report type, date range, and optional project or task scope.
  • Preview row counts and sample data before exporting Excel or CSV.
  • Download past reports again or delete them from the history table.
  • Reports → Insights: see time allocation by project or task and compare periods.
  • Exports use recorded time logs only, never estimates.

Overtime

The overtime page breaks down daily, weekly, and monthly overtime against your workspace targets.

  • Open Overtime from the sidebar while a workspace is active.
  • Review daily overtime hours, weekly bars, and earnings with your workspace multiplier applied.
  • Export an overtime report for the selected month when you need a file copy.

Resources & files

Store documents and attachments alongside your work.

  • Workspace → Resources: browse folders, upload files, import from a folder, and organize with categories.
  • Project → Files: see project files and task attachments in one place.
  • The task drawer and full editor include an attachments section for per-task files.

Notifications

TaskFlow reminds you about scheduled tasks with in-app notifications and optional desktop OS alerts.

  • Use the bell icon in the top bar to open notifications. The badge shows unread count.
  • Task reminders include start date, due soon (3 days, 1 day, 1 hour), and overdue events.
  • Notification rows are grouped into Today and Earlier. Click a row to mark it read and open the task.
  • Mark all read keeps items in the list but clears unread state.
  • Clear all dismisses current notifications from the list.
  • Settings → Notifications lets you enable or disable each reminder type, plus desktop OS alerts.

Backup & restore

Create an encrypted full backup of your database and files on the desktop app. Restore on a new computer or replace data on the same machine. Open Settings → Backup & Restore (desktop app only — not in the browser dev build).

  • Backups are manual. Choose a folder and set a backup passphrase (at least 8 characters). This unlocks the .taskflow-backup file — it is not your app login password.
  • A backup includes all workspaces, projects, tasks, time logs, report records, and local files. It does not include your database password or other security keys stored on this device.
  • Create backup: enter and confirm the backup passphrase, then pick a destination folder. TaskFlow writes a timestamped .taskflow-backup file.
  • Restore on this computer: pick the archive, enter the backup passphrase, and type RESTORE to confirm. This replaces all application data on this install.
  • If you already have a sign-in on this install, restore keeps your current username and password — only data and files are replaced.
  • Optional — restore sign-in: off by default. Only shown when no account exists yet (for example during the first-run wizard on a new PC).
  • New computer: after database setup, the wizard offers restore from backup. Restore data only, then create a new account; or restore sign-in and log in with your old credentials.
  • Forgot your old app password? Restore data only (leave sign-in unchecked), then create a new account in Setup. Your workspaces and tasks return with a new login.
  • Wrong backup passphrase shows an error — nothing is partially restored. Keep your backup passphrase safe; there is no online recovery.

Account & security

Your login protects earnings and reports if someone else uses this computer.

  • Settings → Security: change username or password (current password required).
  • Lock from the account menu or press Ctrl+L to require sign-in again without quitting the app.
  • Enable auto-lock in Settings to lock after idle time (5 to 30 minutes).
  • Logout ends the session completely. Preferences such as theme are kept on this device.
  • There is no online password reset. Keep your password safe. If you forget it, restore from a backup without sign-in and create a new account, or reinstall and clear local app data.

Tips & shortcuts

Small habits that make TaskFlow faster to use.

  • Ctrl+L — lock the app immediately.
  • Week starts on — set under Settings → General; affects weekly progress and reports.
  • Default workspace — set under Settings to skip picking one each session.
  • Tasks URL accepts ?projectId= and ?collectionId= to open a project board (and collection) directly.
